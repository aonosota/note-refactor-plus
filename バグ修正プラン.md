Obsidianコミュニティプラグインの審査基準・実行時の正しさの両面でレビューした結果。
重大バグ3件・中程度のバグ5件・ガイドライン準拠5件。

---

## 重大バグ（必修）

### 1. `pickFile()` が常にキャンセル扱いになる

- **場所**: `src/ui/file-suggester.ts:35`（`onClose`）
- **原因**: Obsidian の `SuggestModal.selectSuggestion` は `this.close()`（→ `onClose()` が同期実行）**の後に** `onChooseItem` を呼ぶ。そのため `onClose` の `reject` が `resolve` より先に走り、Promise は常に reject で確定する。
- **影響**: append-existing プロファイルで対象ファイル空欄（＝実行時選択）のとき、ファイルを選んでもキャンセル扱いになり追記されない。
- **修正**: `onClose` の reject を `setTimeout(..., 0)` で遅延させる。`onChooseItem` は close 直後に同期で呼ばれるため、`chosen` フラグが先に立ち、選択時は reject されなくなる。

### 2. 一括分割（Split by heading）で新規ノートを破壊しうる

- **場所**: `src/core/heading-extractor.ts:285-290`（`doExtract` の afterExtract 処理）
- **原因**:
  - `afterExtract: "open"`（デフォルト値）だと、各見出しの抽出ごとに**同じリーフ**で新規ノートを開く。`editor` インスタンスはそのままドキュメントだけ差し替わるため、次のループの `deleteRange` が**作ったばかりの新規ノート側を削る**。
  - あわせて `"open-new-tab"` が未処理（`"open-new-pane"` → split、それ以外はすべて false）で、新しいタブ設定でも同一リーフで開いてしまう。`src/core/extractor.ts:332` の `openAfterExtract` とは非対称。
- **修正**:
  - `doExtract` にオープン抑止フラグを追加（bulk フラグ `undoStack === null` の流用でも可）し、`splitByHeadingLevel` ではループ中は開かず、全件完了後に必要なら最後の1件だけ開く。
  - リーフ種別の解決を `extractor.ts` の `openAfterExtract` に一本化（export して再利用）し、`"open-new-tab"` → `"tab"` を追加。

### 3. プロファイル編集後、per-profile コマンドが古い設定・古い名前のまま動く

- **場所**: `src/commands.ts:176-190`（`registerProfileCommand`）+ `src/settings/settings-tab.ts:158-169`（`onEdit`）
- **原因**: コマンドのクロージャが **profile オブジェクト自体** を捕捉している。編集保存時は `settings.profiles[i] = saved` で配列要素を新オブジェクトに差し替えるため、コマンドは編集前の設定を参照し続け、リネームもコマンドパレットに反映されない（Obsidian 再起動まで）。
- **修正**:
  - `editorCallback` 内で `getProfileById(plugin.settings, profile.id)` により毎回引き直す（見つからなければ return）。
  - `onEdit` の保存コールバックで `registerProfileCommand(plugin, saved)` を再実行する（同一IDの `addCommand` は上書きされるため名前も更新される）。

## 中程度のバグ

### 4. ファイル名プロンプトのキャンセルがエラー通知になる

- **場所**: `src/core/filename-rule.ts:41-45` + 呼び出し側の catch（`extractor.ts` / `heading-extractor.ts`）
- **症状**: プロンプトを Esc やキャンセルで閉じると `reject(new Error("Cancelled"))` が catch に落ち、「抽出に失敗しました: Cancelled」と表示される。
- **修正**: `resolveFilename` の戻り値を `string | null` にし、キャンセル時は `null` を resolve。呼び出し側は null なら通知なしで中断。

### 5. `onunload` での `saveData`

- **場所**: `src/main.ts:21`
- **問題**: Obsidian ガイドラインで非推奨。アンロード中の非同期保存は完了保証がなく、プラグイン更新時にデータ破損の恐れ。設定は変更のたびに保存済みなので不要。
- **修正**: `onunload` の `saveSettings` を削除。

### 6. append-existing でも保存先フォルダを作成してしまう

- **場所**: `src/core/extractor.ts:36-37`（`extractSelection`）
- **問題**: `resolveDestinationFolder` + `ensureFolder` が append-existing 分岐の**前**にあり、追記モードなのに不要なフォルダが作られる。`splitFromCursor` は分岐後に実行しており非対称。
- **修正**: `extractSelection` でもフォルダ作成を append-existing 分岐の後へ移動。

### 7. 競合モーダルの「追記」でファイル消失時に Promise がハング

- **場所**: `src/core/conflict.ts:63-70`（`promptConflict` の `onAppend`)
- **問題**: `!(abstractFile instanceof TFile)` のとき `return` するだけで resolve されず、コマンドが無言で止まる。
- **修正**: その場合は Notice を出して `resolve({ action: "cancel" })`。

### 8. `splitFromCursor` の行番号スナップショットが await 跨ぎで無効化しうる

- **場所**: `src/core/extractor.ts:126-152`
- **問題**:
  - `totalLines` / `lastLine` をファイル名プロンプト・競合モーダルの**前**に確定しており、モーダル表示中に文書が変わると `replaceRange` の範囲がずれる。
  - ファイル名候補が `lines[0]` 固定のため、カーソル行が空行だと空文字が渡る（`selection.ts` は最初の非空行を使っており非対称）。
- **修正**: `replaceInSource` 内で `editor.lineCount()` を都度取得。ファイル名候補は最初の非空行を使う。

## ガイドライン準拠・軽微

### 9. 設定モーダルの `createEl("h3")`

- **場所**: `src/settings/profile-editor.ts:59-61`（`sectionHeading`）
- 審査基準は見出しタグ直書きではなく `new Setting(el).setName(...).setHeading()` を推奨（settings-tab.ts では対応済み）。CSS の `.nrp-section-heading` は Setting の heading 要素へ付け替え。

### 10. インラインスタイル `el.style.display = ...`

- **場所**: `src/settings/profile-editor.ts`（5箇所: pattern / fixedPath / appendDetails / headingName の表示切替）
- 審査Botが「Avoid setting inline styles」でフラグする定番。`styles.css` に `.nrp-hidden { display: none; }` を追加し `toggleClass("nrp-hidden", cond)` へ置換。

### 11. `types.ts` の doc コメントが実装と不一致

- **場所**: `src/types.ts:43`
- `noteLinkTemplate` の変数として `{{link}}` と記載されているが、実際に展開されるのは `{{source_link}}`（`src/core/template.ts`）。コメント修正。

### 12. パターン切替時にプレースホルダ文字列が実値として入る

- **場所**: `src/settings/profile-editor.ts:194-227`（`renderFilenameSection`）
- mode を "pattern" に切り替えると `t("profile.pattern-placeholder")` が pattern の実値としてセットされ、そのまま保存されうる（「パターン未入力」バリデーションも実質機能しない）。
- **修正**: 初期値は空にし、例示は `setPlaceholder` のみで行う。

### 13. `package.json` の `"obsidian": "latest"`

- ビルド再現性のため固定バージョンにピン留め推奨（minAppVersion 1.13.1 に対応する型定義バージョン）。

- `metadataCache` の見出し位置がエディタの未保存編集と乖離しうる件（heading-extractor 全般）。再現条件が限定的なためフォローアップ扱い。
- `main.js` をリポジトリにコミットしている件（BRAT運用なら意図的）。
- Note Refactor インポートの `overwriteExistingNote` → `"append"` マッピングの意味差（元は上書き）。
